1. Introduction  

Groff (known as “GNU Troff” ) is traditional Unix system (from ’70) to generate documentaion. Most Unix Manual pages are still written using “Groff”.

Originally Groff was used mainly for terminal output (including many backspace sequences, for example for highlight) and PostScript output - to create hard-copy of manuals.

2. Using groff  

Tested on Debian10.

2.1. System preparation  

At first we should install following packages:

sudo apt-get install git make groff

Now checkout this sample project using:

mkdir -p ~/projects
cd ~/projects
git clone https://github.com/hpaluch/html-and-pdf-generators.git
cd html-and-pdf-generators

2.2. Invoking groff  

To generate PDF and HTML output do this:

cd ~/projects/html-and-pdf-generators/groff/
make

There should be no warning and/or error reported and these new files should be generated:

GroffPoC.pdf

PDF outptut for GroffPoC.me input file

GroffPoC/index.html

HTML output for GroffPoC.me input file

And that’s all so far!
